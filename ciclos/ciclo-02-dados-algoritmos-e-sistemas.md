# Ciclo 2 — Dados, Algoritmos e Sistemas

O segundo ciclo aproxima os fundamentos de problemas computacionais mais estruturados. Depois de compreender vocabulário, lógica, representação e execução, o aluno passa a estudar como dados são organizados, processados, persistidos, consultados, integrados e mantidos dentro de sistemas.

Este ciclo é decisivo para a futura compreensão de sistemas corporativos. Mainframe e COBOL lidam intensamente com registros, arquivos, bancos de dados, algoritmos de negócio, integrações e manutenção. Esses assuntos precisam ser entendidos antes de serem encontrados em uma tecnologia específica.

## Fases pertencentes ao ciclo

- Fase 7 — Estruturas de dados
- Fase 8 — Algoritmos corporativos
- Fase 9 — Arquivos e persistência
- Fase 10 — Bancos de dados
- Fase 11 — Redes e integração
- Fase 12 — Engenharia de software

## Base que o aluno deve construir

Ao final deste ciclo, o aluno deve compreender que a forma de organizar dados influencia o processamento. Deve saber distinguir arquivo de banco de dados, entender por que integrações precisam de contratos e perceber que software possui ciclo de vida, manutenção, documentação e risco.

O aluno deve começar a enxergar sistemas como combinações de dados, regras, fluxos, interfaces e responsabilidades. Essa visão será necessária para entender por que ambientes corporativos exigem mais disciplina do que exemplos isolados de programação.

## Por que este ciclo vem antes do proximo

O Ciclo 3 tratará de sistemas corporativos, processamento batch e Mainframe. Esses temas pressupõem que o aluno já saiba o que são arquivos, bancos de dados, algoritmos de negócio, integração e engenharia de software. Sem essa base, o Mainframe pareceria apenas uma plataforma diferente, e não uma resposta a problemas específicos de operação e dados.

Este ciclo cria a ponte entre computação geral e computação corporativa. Ele prepara o aluno para entender volume, confiabilidade, consistência, rastreabilidade e continuidade.

## Erros de aprendizado que este ciclo evita

Este ciclo evita que o aluno trate dados como valores soltos sem estrutura. Também evita a ideia de que sistemas são apenas telas e códigos, ignorando persistência, contratos, integrações, manutenção e qualidade.

Outro erro que este ciclo combate é imaginar que algoritmos são apenas desafios abstratos. Na formação, algoritmos também serão estudados como formas de totalizar, conciliar, validar, classificar e transformar dados de negócio.
