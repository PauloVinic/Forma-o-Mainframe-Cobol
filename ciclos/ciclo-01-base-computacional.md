# Ciclo 1 — Base Computacional

O primeiro ciclo estabelece a base conceitual da formação. Ele existe para que o aluno compreenda o que significa estudar computação antes de entrar em linguagens, ferramentas, ambientes de execução ou tecnologias corporativas.

Neste ciclo, a atenção está em vocabulário, raciocínio, lógica, representação de dados, arquitetura de computadores e sistemas operacionais. Esses temas não são uma introdução decorativa; eles sustentam tudo o que virá depois. Sem eles, Mainframe e COBOL seriam apresentados como um conjunto de comandos sem contexto.

## Fases pertencentes ao ciclo

- Fase 0 — Alfabetização técnica
- Fase 1 — Pensamento computacional
- Fase 2 — Lógica de programação sem linguagem
- Fase 3 — Matemática discreta aplicada
- Fase 4 — Representação de dados
- Fase 5 — Arquitetura de computadores
- Fase 6 — Sistemas operacionais

## Base que o aluno deve construir

Ao final deste ciclo, o aluno deve conseguir usar termos técnicos básicos com mais precisão, decompor problemas simples, reconhecer entradas e saídas, entender decisões e repetições, perceber que dados precisam ser representados e compreender que programas executam dentro de limites definidos por hardware e sistema operacional.

O aluno não precisa ainda escrever programas COBOL, operar um mainframe ou executar jobs. O objetivo é formar a estrutura mental que permitirá entender essas práticas quando elas aparecerem.

## Por que este ciclo vem antes do proximo

O Ciclo 2 tratará de estruturas de dados, algoritmos, arquivos, bancos, redes e engenharia de software. Esses assuntos dependem diretamente da base construída aqui. Não há como discutir persistência, processamento, integração ou manutenção sem antes entender dados, execução, lógica e recursos computacionais.

Este ciclo também cria um ritmo de estudo. O aluno aprende que cada conceito deve ser compreendido em camadas, não apenas memorizado.

## Erros de aprendizado que este ciclo evita

Este ciclo evita que o aluno confunda programação com digitação de sintaxe. Também evita que termos como memória, arquivo, sistema, dado, programa e processamento sejam usados de modo vago.

Outro erro comum é tentar aprender COBOL como se fosse apenas uma linguagem antiga com comandos diferentes. A base computacional mostra que a linguagem só fará sentido quando estiver ligada a dados, arquivos, processamento e ambiente de execução.
