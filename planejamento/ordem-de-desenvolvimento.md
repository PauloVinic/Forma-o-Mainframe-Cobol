Nenhuma fase futura deve ser desenvolvida antes da anterior estar minimamente coerente. Isso não significa que uma fase precise estar perfeita para que o projeto avance, mas ela deve ter objetivo, escopo, conteúdos principais, critérios de domínio e conexão com a etapa seguinte.

## Como decidir se uma fase pode avancar

Uma fase pode ser considerada minimamente coerente quando:

- o objetivo da fase está claro;
- o escopo não invade fases futuras;
- os conceitos principais foram explicados com profundidade suficiente;
- os termos novos foram definidos;
- há conexão com as fases anteriores e posteriores;
- o conteúdo respeita as diretrizes editoriais;
- existem critérios de domínio para avaliar aprendizagem.

Se esses elementos não estiverem presentes, o próximo prompt deve revisar a fase atual em vez de criar a fase seguinte.

## Atualizacao do planejamento

Quando uma fase for desenvolvida, os documentos de planejamento podem ser atualizados se surgirem ajustes de escopo ou novas dependências. Essas atualizações devem ser pontuais e justificadas. O roadmap não deve ser reescrito a cada etapa sem necessidade.
