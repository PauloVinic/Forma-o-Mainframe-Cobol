# Prompt Inicial — Regras do Projeto

Este arquivo registra uma versão resumida do prompt inicial usado para criar a fundação do projeto `formacao-cs-mainframe-cobol`.

## Objetivo solicitado

Criar a fundação organizacional, pedagógica e editorial de uma formação progressiva em Ciência da Computação voltada para Mainframe e COBOL.

O projeto não deveria começar por COBOL, ferramentas, instalação de ambiente ou exemplos práticos rápidos. A formação deveria começar pelos fundamentos: computação, pensamento computacional, lógica, dados, arquitetura de computadores, sistemas operacionais, estruturas de dados, algoritmos, arquivos, bancos de dados, redes, engenharia de software, sistemas corporativos, processamento batch, Mainframe e somente depois COBOL, JCL, VSAM, DB2, CICS e modernização.

## Limites definidos

O prompt determinou que, neste primeiro passo, não fossem criadas aulas completas, código, Dockerfile, Makefile, exemplos COBOL ou módulos inteiros.

A tarefa foi criar apenas a estrutura inicial do projeto e os documentos de orientação:

- README.md;
- ROADMAP.md;
- diretrizes do projeto;
- diretrizes editoriais;
- manifesto pedagógico;
- guia de uso dos prompts;
- glossário inicial;
- documentos dos ciclos;
- templates;
- área de prompts;
- área de referências;
- documentos de planejamento.

## Regras editoriais essenciais

O conteúdo deve ser escrito em português do Brasil, com clareza, sobriedade e profundidade suficiente. Não deve usar linguagem promocional, não deve virar lista superficial e não deve simular completude por volume de arquivos.

As futuras aulas devem ser escritas como capítulos de estudo, com explicações desenvolvidas, exemplos contextualizados, transições naturais e conexão com sistemas reais, Mainframe ou COBOL quando isso fizer sentido.

## Proxima etapa indicada

A próxima etapa recomendada é desenvolver a Fase 0 — Alfabetização técnica, ainda sem criar código, exercícios práticos ou conteúdo de COBOL.
