# Prompt 003 — Aula 02: O que é um sistema

Este arquivo registra uma versão resumida do prompt usado para desenvolver a Aula 02 da Fase 0 — Alfabetização Técnica.

## Objetivo Solicitado

Criar somente a aula completa `Aula 02 — O que é um sistema`, no caminho `ciclo-01-base-computacional/fase-00-alfabetizacao-tecnica/aulas/aula-02-o-que-e-um-sistema.md`.

A aula deveria funcionar como capítulo de estudo, explicando sistema como conceito fundamental para entender tecnologia da informação, sistemas corporativos, sistemas computacionais, sistemas legados, Mainframe e COBOL futuramente.

## Documentos Que Deveriam Ser Lidos

O prompt exigiu leitura dos documentos globais do projeto, das diretrizes editoriais, do manifesto pedagógico, dos critérios de qualidade, da ordem de desenvolvimento, do template de aula, dos documentos de planejamento da Fase 0 e da Aula 01 — O que é tecnologia da informação.

## Conteúdo Central da Aula

A aula deveria explicar por que o conceito de sistema é essencial antes de computação e programação, o que significa dizer que algo é um sistema, a diferença entre conjunto de partes e sistema, a relação entre componentes, organização, propósito, fronteira, ambiente, interfaces, entrada, processamento, saída e retroalimentação.

Também deveria abordar tipos de sistemas, empresas como sistemas, processos de negócio como sistemas, sistemas de informação, sistemas corporativos, falhas em componentes, dependências entre partes e conexão futura com Mainframe e COBOL sem ensinar esses temas tecnicamente.

## Limites Definidos

O prompt determinou que não fossem criadas Aula 03, exercícios completos, avaliações, código, COBOL, Dockerfile, Makefile ou ambiente de desenvolvimento. Também determinou que a aula não fosse escrita como resumo, lista superficial ou conteúdo promocional.

## Atualizações Solicitadas

Além da criação da aula, o prompt solicitou atualização do README da pasta `aulas/`, do README da Fase 0 e de `planejamento/ordem-de-desenvolvimento.md`, registrando que a próxima etapa será desenvolver somente a Aula 03 — Hardware, software e pessoas.

## Próxima Etapa Recomendada

A próxima etapa recomendada é desenvolver somente a Aula 03 — Hardware, software e pessoas, seguindo o mesmo padrão editorial, sem criar código, exercícios completos, avaliações ou conteúdo de COBOL.
