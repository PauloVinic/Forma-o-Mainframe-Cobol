# Glossario Inicial

Este glossário reúne termos fundamentais que aparecerão ao longo da formação. As definições são iniciais e serão aprofundadas nas fases correspondentes.

## Computacao basica

**Computador**: sistema capaz de receber dados, armazená-los, processá-los segundo instruções e produzir resultados. A ideia central não é apenas a máquina física, mas a combinação entre componentes, programas, dados e uso.

**Hardware**: conjunto de componentes físicos que participam do funcionamento de um computador, como processador, memória, dispositivos de armazenamento e interfaces de entrada e saída. Ele importa porque todo software depende de recursos físicos para executar.

**Software**: conjunto de instruções, programas e dados organizados para orientar o funcionamento do computador. Software não é apenas aplicativo visível ao usuário; inclui sistemas operacionais, utilitários, bibliotecas e programas internos.

**Programa**: sequência organizada de instruções que descreve uma tarefa a ser executada. Um programa expressa uma intenção de processamento, mas só produz efeito quando executado em um ambiente capaz de interpretá-lo ou compilá-lo.

**Algoritmo**: descrição ordenada de passos para resolver um problema. Antes de existir em uma linguagem de programação, um algoritmo pode ser pensado como raciocínio estruturado sobre entradas, decisões, transformações e saídas.

**Abstracao**: forma de lidar com complexidade escondendo detalhes temporariamente. A abstração permite estudar uma parte do sistema sem precisar carregar todos os detalhes físicos, lógicos e operacionais ao mesmo tempo.

**Interface**: ponto de contato entre partes de um sistema, como usuário e aplicação, programa e arquivo, aplicação e banco de dados ou sistemas distintos. Uma boa interface deixa claro o que pode ser enviado, recebido e esperado.

## Dados

**Dado**: registro bruto de alguma coisa observada, informada ou armazenada. Um dado só ganha utilidade quando seu formato, significado e contexto são compreendidos.

**Informacao**: dado interpretado dentro de um contexto. Um número isolado pode ser apenas dado; quando se sabe que ele representa saldo, idade, código de cliente ou quantidade de parcelas, ele passa a carregar significado.

**Bit**: menor unidade lógica de representação digital, com valor 0 ou 1. Bits são a base para representar números, caracteres, instruções e estados em sistemas computacionais.

**Byte**: grupo de bits usado como unidade comum de armazenamento e endereçamento. Em muitos contextos, um byte é associado à representação de caracteres ou pequenas quantidades de informação binária.

**Tipo de dado**: classificação que indica como um valor deve ser interpretado e manipulado, como número, texto, data ou indicador lógico. Tipos ajudam a evitar confusões entre forma de armazenamento e significado.

**Campo**: parte específica de um registro, destinada a guardar um valor com significado próprio. Em um cadastro, por exemplo, nome, data de nascimento e código do cliente podem ser campos.

**Registro**: conjunto estruturado de campos relacionados a uma mesma entidade ou ocorrência. Muitos arquivos corporativos são compostos por registros que seguem um layout definido.

**Chave**: valor usado para identificar, localizar, ordenar ou relacionar registros. Chaves são importantes em arquivos, bancos de dados, integrações e controles de consistência.

## Processamento

**Entrada**: dado recebido por um sistema para ser processado. A entrada pode vir de uma pessoa, arquivo, banco de dados, mensagem, transação ou outro sistema.

**Saida**: resultado produzido por um processamento. Pode ser uma tela, relatório, arquivo, atualização em banco de dados, mensagem para outro sistema ou registro de log.

**Processamento**: transformação de entradas em saídas por meio de regras, cálculos, decisões e movimentações de dados. Em sistemas corporativos, o processamento costuma refletir regras de negócio.

**Memoria**: recurso usado para manter dados e instruções acessíveis durante a execução. A memória é geralmente mais rápida que o armazenamento permanente, mas seu conteúdo pode ser temporário.

**Armazenamento**: meio usado para preservar dados depois que o processamento termina. Arquivos, bancos de dados e mídias de armazenamento existem para manter informação além da execução imediata.

**Entrada e saida**: operações pelas quais um sistema lê dados de fora do processamento imediato ou grava resultados para uso posterior. Entrada e saída são centrais para arquivos, bancos, telas, redes e dispositivos.

**Batch**: forma de processamento em lote, na qual um conjunto de dados é processado sem interação individual imediata com cada usuário. Rotinas batch são comuns em fechamentos, conciliações, faturamentos e atualizações massivas.

**Transacao**: unidade de trabalho que representa uma operação relevante, como consultar saldo, efetuar pagamento ou alterar cadastro. Em sistemas corporativos, transações precisam de consistência, controle e rastreabilidade.

## Sistemas

**Sistema operacional**: software responsável por gerenciar recursos do computador, como processos, memória, arquivos, dispositivos e permissões. Ele cria as condições para que programas executem de forma controlada.

**Processo**: programa em execução, com recursos associados e estado próprio. Entender processos ajuda a compreender execução, concorrência, falhas e consumo de recursos.

**Arquivo**: coleção persistente de dados organizada para armazenamento e leitura posterior. Arquivos podem guardar textos, registros estruturados, configurações, relatórios ou dados intermediários de processamento.

**Banco de dados**: sistema organizado para armazenar, consultar, relacionar e proteger dados. Bancos de dados são fundamentais quando a informação precisa ser consistente, compartilhada e recuperável.

**Rede**: meio de comunicação entre computadores e sistemas. Redes permitem integração, acesso remoto, troca de mensagens e funcionamento de sistemas distribuídos.

**Log**: registro de eventos relevantes ocorridos em um sistema. Logs ajudam investigação, auditoria, diagnóstico de falhas e reconstrução de comportamento.

**Disponibilidade**: capacidade de um sistema permanecer acessível quando necessário. Em sistemas críticos, indisponibilidade pode afetar operações financeiras, atendimento, produção ou obrigações legais.

**Seguranca**: conjunto de práticas e mecanismos para proteger dados, acessos, operações e infraestrutura. Segurança envolve confidencialidade, integridade, disponibilidade e controle de permissões.

## Sistemas corporativos

**Regra de negocio**: condição, cálculo, restrição ou procedimento que representa uma decisão da organização. Em sistemas corporativos, muitas partes do código existem para aplicar regras de negócio.

**Sistema corporativo**: sistema usado para sustentar processos de uma organização, como cadastro, faturamento, contabilidade, estoque, pagamento ou atendimento. Esses sistemas costumam ter dados importantes e vida longa.

**Sistema critico**: sistema cuja falha pode causar impacto relevante para pessoas, operação, dinheiro, reputação ou obrigações legais. O estudo de sistemas críticos exige atenção a confiabilidade e controle.

**Sistema legado**: sistema antigo ou consolidado que continua sendo importante para a organização. Legado não significa inútil; muitas vezes significa que o sistema concentra regras, histórico e dependências relevantes.

**Auditoria**: capacidade de verificar o que ocorreu, quando ocorreu, quem participou e quais dados foram afetados. Auditoria é essencial em ambientes regulados e em processos que exigem prestação de contas.

**Rastreabilidade**: possibilidade de acompanhar a origem, transformação e destino de dados ou decisões. Ela permite investigar problemas e demonstrar como um resultado foi produzido.

**SLA**: acordo ou meta de nível de serviço, geralmente relacionado a disponibilidade, tempo de resposta, recuperação ou atendimento. Em sistemas corporativos, SLAs influenciam arquitetura, operação e suporte.

**Integracao**: comunicação ou troca de dados entre sistemas. Integrações exigem contratos claros, formatos definidos, tratamento de erro e compreensão das dependências entre processos.

## Mainframe

**Mainframe**: plataforma computacional projetada para processar grandes volumes de dados e transações com alta confiabilidade. Seu valor está ligado a operação contínua, controle, escalabilidade e histórico corporativo.

**z/OS**: sistema operacional usado em ambientes IBM Z. Ele gerencia recursos, execução de trabalhos, arquivos, segurança, transações e serviços necessários ao funcionamento do ambiente mainframe.

**Dataset**: forma comum de organizar dados em z/OS. Um dataset pode ser entendido inicialmente como uma estrutura persistente usada por programas, jobs e utilitários para ler e gravar dados.

**Job**: unidade de trabalho submetida para execução em ambiente mainframe, especialmente em processamento batch. Um job descreve etapas, programas, entradas, saídas e recursos necessários.

**JCL**: linguagem usada para descrever jobs em ambientes z/OS. Ela não é uma linguagem de programação de negócio, mas uma forma de orientar o sistema operacional sobre como executar trabalhos.

**Spool**: área de gerenciamento de entradas e saídas de jobs, relatórios e mensagens. O spool ajuda a controlar execução, impressão lógica, resultados e diagnóstico operacional.

**VSAM**: método de acesso e organização de dados muito usado em mainframe. Ele é importante para arquivos estruturados com necessidades específicas de acesso, chave e desempenho.

**DB2**: sistema gerenciador de banco de dados relacional usado em muitos ambientes corporativos e mainframe. Ele permite organizar dados em tabelas, consultar informações e preservar consistência transacional.

**CICS**: ambiente de processamento de transações online em mainframe. Ele permite que programas atendam operações interativas com controle, desempenho e integração com dados corporativos.

## COBOL e tecnologias futuras

**COBOL**: linguagem criada para expressar regras de negócio e processamento de dados de forma legível e estruturada. Na formação, COBOL será estudado depois da base necessária para entender seu uso em sistemas reais.

**Divisao COBOL**: grande seção estrutural de um programa COBOL. O estudo das divisões será feito futuramente, quando o aluno já entender a relação entre identificação, ambiente, dados e procedimentos.

**Copybook**: trecho reutilizável de definição ou código em COBOL, frequentemente usado para padronizar layouts de dados. Copybooks são importantes para consistência entre programas que compartilham estruturas.

**Arquivo sequencial**: arquivo lido ou gravado em ordem, registro após registro. Esse modelo é fundamental para entender muitos processamentos batch e rotinas corporativas.

**Manutencao corretiva**: alteração feita para corrigir erro identificado em um sistema. Em sistemas corporativos, exige cuidado para não quebrar regras ou fluxos dependentes.

**Manutencao evolutiva**: alteração feita para adaptar o sistema a novas necessidades de negócio. Ela exige compreensão do funcionamento atual antes de introduzir mudanças.

**Modernizacao**: processo de atualizar, integrar ou reorganizar sistemas existentes sem perder regras e dados essenciais. Modernizar não é simplesmente substituir tecnologia; é preservar valor enquanto se reduz risco e limitação.
